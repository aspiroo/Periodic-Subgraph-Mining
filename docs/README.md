## Documentation Standards

All documentation follows these principles:

1. **Practical**: Step-by-step instructions
2. **Clear**: Plain language, avoid jargon
3. **Complete**: Cover all necessary information
4. **Tested**: Commands that actually work
5. **Current**: Updated with repository changes
